# Codex Prompt Template

## 目的

- [今回の 1 目的を簡潔に書く]

## 今回やること

- [最小差分でやる実装・修正・整理]
- [必要なら補助的な変更]
- [必要なら確認やテスト]

## 制約

- 日本語で進める
- 1 フェーズ = 1 目的に限定する
- 差分中心で進める
- 既存動作を壊さない
- 過剰な大改造をしない
- 特定案件の未確認前提に依存しすぎない
- 必要な更新対象だけを触る

## 期待する進め方

- まず関連ファイルを確認する
- 必要な実装・修正を行う
- 必要なら最小限のテストや確認を行う
- 完了時は `bridge/outbox/codex_report.md` に完了報告を書く

## 確認項目

- 変更範囲が今回の目的に収まっている
- 既存の主要フローを壊していない
- 追加した確認内容を説明できる
- 未対応事項があれば注意点に明記する

## 最終報告形式

以下の 6 項目固定で、日本語で簡潔にまとめること。

1. 実施概要
2. 変更した主要ファイル
3. 実装内容 / 仕様
4. 確認結果
5. --no-ff マージ・push結果
6. 注意点
