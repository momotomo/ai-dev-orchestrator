# Codex Prompt Sample

## 目的

- 既存の export まわりの使い勝手を壊さずに、エラーメッセージと完了表示をわかりやすくする

## 今回やること

- export 処理の失敗時メッセージを整理する
- 成功時の結果表示を簡潔に改善する
- 関連する文言や補助テストが必要なら最小限で追加する

## 制約

- 日本語で進める
- 1 フェーズ = 1 目的に限定する
- 差分中心で進める
- 既存動作を壊さない
- 過剰な大改造をしない
- UI 全面改修や構成変更には広げない

## 期待する進め方

- まず export 関連の既存実装と表示箇所を確認する
- 影響範囲を絞って修正する
- 確認できる範囲でテストまたは動作確認を行う
- 完了時は `bridge/outbox/codex_report.md` に完了報告を書く

## 確認項目

- export 成功時と失敗時の表示が整理されている
- 既存フローを壊していない
- テスト追加や未実施確認があれば明記する

## 最終報告形式

以下の 6 項目固定で、日本語で簡潔にまとめること。

1. 実施概要
2. 変更した主要ファイル
3. 実装内容 / 仕様
4. 確認結果
5. --no-ff マージ・push結果
6. 注意点

## 別テーマにも流用しやすい例

- UI polish
- テスト追加
- 小さめのリファクタ
- ドキュメント整理
